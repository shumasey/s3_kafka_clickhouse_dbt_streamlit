# Лаба 8 Проект - Транзакционная аналитика

Данные поступают из двух источников одновременно:
- **Батч-данные на S3** — каждые 10 минут появляется новый файл с транзакциями за последний интервал
- **Стрим событий в Kafka** — те же данные, но в реальном времени, событие за событием
Аналитический пайплайн забирает данные из обоих источников, складывает в хранилище, считает метрики и показывает дашборд с визуализациями.

В качестве хранилища используется **ClickHouse** учебного центра, поэтому его развертывание не входит в этот проект. Батчевые данные обрабатываются с помощью **DBT**, обработка стриминга реализована на ClickHouse. Подготовленные витрины демонстрирует приложение **Streamlit**.

Данные в формате JSON напрямую считываются из S3 в исходные таблицы ClickHouse. 
Из исходных таблиц DBT осуществляет инкрементальную загрузку в слой RAW, добавляя временную метку загрузки.
Из слоя RAW DBT загружает данные в слой CORE, по пути приводя временные данные к единому виду, заполняя пустоты и добавляя полезные поля с датой, часами, временем завершения действия курсов валют.
Из слоя CORE DBT создает витрину для дашборда в слое DM.
