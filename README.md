# Лаба 8 Проект - Транзакционная аналитика

Данные поступают из двух источников одновременно:
- **Батч-данные на S3** — каждые 10 минут появляется новый файл с транзакциями за последний интервал
- **Стрим событий в Kafka** — те же данные, но в реальном времени, событие за событием
Аналитический пайплайн забирает данные из обоих источников, складывает в хранилище, считает метрики и показывает дашборд с визуализациями.

В качестве хранилища используется **ClickHouse** учебного центра, поэтому его развертывание не входит в этот проект. Батчевые данные обрабатываются с помощью **DBT**, обработка стриминга реализована на ClickHouse. Подготовленные витрины демонстрирует приложение **Streamlit**.

![pipe_schema.jpg](/img/pipe_schema.jpg)

## DBT and Clickhouse
1. Данные в формате JSON напрямую считываются из S3 в исходные таблицы ClickHouse. 
2. Из исходных таблиц DBT осуществляет инкрементальную загрузку в слой RAW, добавляя временную метку загрузки.
3. Из слоя RAW DBT загружает данные в слой CORE, по пути приводя временные данные к единому виду, заполняя пустоты и добавляя полезные поля с датой, часами, временем завершения действия курсов валют.
4. Из слоя CORE DBT создает витрину для дашборда в слое DM.


## Kafka and Clickhouse
После создания табличек на основе dbt, что собирают по расписанию s3 мы создадим таблицы, materialized view к ним для соединения батч и стрим слоя, дедупликации и обогащения данными справочниками и аггрегациями.

Для этого перейдем в папку kafka_click_table_creation и пропишем, не забыв заменить креды на свои
```
chmod +x click_table_creation
./click_table_creation
```
После этого создадутся таблицы пригодные для использования в дашборде 

## Streamlit

Задать переменные окружения click_user, click_port, click_pass
Затем запускаем streamlit

```
cd streamlit
chmod +x run_streamlit
./run_streamlit
```

После этого на 8501 развернется streamlit, открываем через браузер http://i.p.add.ress:8501

# Запуск проекта
Проект собирается в контейнер автоматически.
1. Подготовьте ClickHouse для хранилища.
2. В ОС Windows используйте WSL, в Linux просто выполните команду
```
docker run -d \
  --name s3_kafka_dbt \
  -p 8501:8501 \
  -v "$(pwd)/profiles.yml:/root/.dbt/profiles.yml" \
  -e CLICK_HOST="your.ch.ip.address" \
  -e CLICK_USER="your_user_name" \
  -e CLICK_PASS="user_name_password" \
  -e CLICK_DB="your_ch_schema" \
  -e CLICK_PORT="8123" \
  shumasey/s3_kafka_clickhouse_dbt_streamlit:latest \
  bash -c "streamlit run /app/streamlit/dashboard.py --server.port=8501 --server.address=0.0.0.0 --server.headless True & while true; do echo 'Обновляю витрину данных...'; dbt run --select +lab08_dm_base_common; sleep 600; done"
```
3. В браузере зайдите на сайт ```localhost:8501```
